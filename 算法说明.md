# KOC-Engine 底层算法说明

KOC-Engine 当前后端实现的是一条可在 Vercel Serverless 上运行的轻量级可解释分析管线。它不依赖本地视频解码或长耗时模型推理，而是把内容链接、页面元信息和文本语义转化为可解释特征，再生成雷达分、留存曲线、干预节点和脚本重构建议。

## 输入层

1. URL 结构解析：识别平台域名、路径、查询参数、分享标题、分享描述。
2. 页面元信息抓取：在 2.6 秒超时内读取 HTML，并解析 OpenGraph / Twitter Card / meta description / title / cover image。
3. 降级融合：如果页面失效、反爬或返回低价值标题，例如“视频不见了”，系统会优先使用分享参数和 URL 语义。

## 特征工程

系统从文本中提取以下特征：

- `hook_strength`：由 Hook 触发词、标题压缩度、产品词、平台先验共同估计。
- `proof_strength`：由实测、对比、截图、数据、反馈等证据词，以及 OpenGraph 元信息置信度共同估计。
- `cta_strength`：由链接、领取、库存、评论、购买、收藏等行动词估计。
- `emotion_variance`：由情绪词、反差表达、URL 语义纹理估计。
- `content_density`：由标题长度、描述长度、元信息完整度估计。
- `scene_transfer`：由校园、宿舍、通勤、军训等场景词估计可迁移性。

## 五维雷达

五维分数不是随机数，来自规则线性组合：

- 视觉张力：内容密度、Hook 强度、元信息置信度、封面图信号。
- BGM 契合：情绪方差、内容密度、平台先验。
- 前 3 秒留存：Hook 强度、标题压缩度、Hook 词命中。
- 情绪波动：情绪词、证据强度、反差表达。
- Hook 密度：Hook 强度、CTA 强度、证据强度、产品词命中。

为避免同类内容输出完全一致，系统使用 URL hash 生成极小稳定扰动。同一个链接多次解析结果保持稳定，不同链接结果会有差异。

## 60 秒留存曲线

留存曲线由三部分组成：

1. 初始留存：受 Hook 强度影响。
2. 衰减基线：按 0-60 秒逐步下降。
3. 局部反弹：证据补强点、情绪波峰点、CTA 收口点会形成小幅回升。

输出固定为 60 个浮点数，供前端绘制留存波形和干预节点。

## 脚本 Diff

系统会生成：

- 原始脚本：模拟普通创作者的低效率表达。
- 优化脚本：根据 Hook、证据、场景、CTA 四类阻力重构。
- 逐句 Diff：每行包含原句、问题标签、优化句和改写原因。

## LLM 增强层

默认情况下，基础分数、曲线和干预节点完全由可解释特征工程生成。环境变量 `KOC_ENGINE_ENABLE_OPENAI=1` 时，系统会调用 OpenAI 兼容接口，只润色文案和 insight 表达，不改动核心分数。

## Vercel 约束

该实现避免使用 `yt-dlp`、`opencv`、`moviepy` 等重依赖，保证 Serverless 冷启动、执行时长和包体可控。外链抓取失败时，系统仍可通过 URL 与分享参数完成解析。
