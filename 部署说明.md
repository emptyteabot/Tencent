# KOC-Engine 部署说明

## 本地运行

前端：

```bash
npm install
npm run dev
```

后端：

```bash
uvicorn main:app --reload --port 8000
```

本地前端会请求 `http://localhost:8000/api/reverse-engineer`。

## API 配置优先级

后端读取顺序如下：

1. 环境变量 `OPENAI_API_KEY`
2. 本机 `C:\Users\cyh\.codex\auth.json` 中的 `OPENAI_API_KEY`
3. 环境变量 `OPENAI_BASE_URL`
4. 本机 `C:\Users\cyh\.codex\config.toml` 中的 `base_url`
5. 环境变量 `OPENAI_MODEL`
6. 本机 `C:\Users\cyh\.codex\config.toml` 中的 `model`

也就是说，这个项目默认就能复用你本地 Codex CLI 的 OpenAI 兼容配置。

## Vercel 部署

导入项目根目录后，设置以下环境变量：

```bash
OPENAI_API_KEY=你的兼容 Key
OPENAI_BASE_URL=你的 OpenAI 兼容接口地址
OPENAI_MODEL=gpt-5.4
OPENAI_TIMEOUT_SECONDS=60
```

如果你只是路演，不想真的调模型，可以加：

```bash
KOC_ENGINE_FORCE_MOCK=1
```

这样线上也会稳定返回高质量演示数据，不受上游波动影响。

## 已实现的兜底

- 上游模型超时或失败时，后端自动返回 Mock JSON
- 前端请求失败时，还会再退回一层本地演示数据
- 因此页面不会空白，按钮点下去一定有结果
